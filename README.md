An LSTM-based trend prediction system for swing trading with feature engineering.
## 專案目標:
本專案是開發應用於 **短期波段交易(Short-term Swing Trading)** 的預測系統，核心目標是利用深度學習技術，從複雜與充滿雜訊的市場波動中篩選出 **極高勝率** 的短線進場點，解決一般投資者在短線操作時容易而誤判的問題。
## 核心設計理念:
本專案的開發邏輯並非追求頻繁交易，而是追求極低的 **犯錯率（False Alarm Rate）** 與高信心度的選點，從而達成「高期望值」的交易策略：
### 模型架構:
採用 **LSTM (Long Short-Term Memory)** 深度學習架構，由於股價具備強烈的時序依賴性，LSTM 擅長捕捉時間序列中的非線性特徵與短期動能轉折，比傳統技術指標更具備預測能力。
### 特徵工程:
* 交易量動能:
計算當日成交量與 14 日均量的比值，判斷短期上漲是否具備實質資金支撐。
* 多方信心程度:
統計過去 14 日內紅棒出現的頻率，量化市場短期的多方慣性與心理偏向。
* 相對強弱強度:
衡量股價偏離均線的標準差倍數，藉此判斷動能的異常與否，識別具備統計意義的突破點。
* K 棒實體強度: 
計算 K 棒實體佔全日波動的比例，有效過濾上下影線過長的震盪盤。
### 標籤邏輯:
本專案不採用簡單的「隔日漲跌」作為標籤，而是設計了更符合實戰穩定性的「波段濾波標籤」
* 核心公式：label = (未來3日收盤均價 > 14日均線) AND (未來第3日收盤價 > 14日均線)
* 設計理念：
  * 採用收盤價進行回測與標記，是因為接近盤後交易時價格波動相對穩定，能減少實務操作中的滑價誤差。
  * 透過未來三日均價的判定，過濾掉單日虛增但隨即反轉、或難以實際成交的極端波動。
  * 增加 **未來第3日收盤價** 作為限制條件，是為了確保該進場點處於明確的上升趨勢中。

### 回測與優化:
基於 **元大50 (0050)** 數據，進行 10 次獨立訓練之平均表現：

**數據集與回測時程**：
* **訓練集**: `2020-01-01` 至 `2025-12-31`
* **測試集**: `2026-01-01` 至 `2026-05-11`

**參數優化紀錄**：
* 經多輪迭代實驗，最終採用長時訓練策略（600 Epochs）配合低學習率（1e-4），以確保模型在高度雜訊的金融環境中穩定收斂。
* 特別針對金融時間序列特性，我們將 DataLoader 的 shuffle 設為 **False**，嚴格限制模型僅能依循時序規律學習。

**交易策略與風險管理**：
* **進場門檻：** 模型預測機率高於五成時進場。
* **停損機制：** 若價格低於進場價 1%，立即觸發停損。
* **停利機制：** 若價格高於進場價 3%，執行獲利了結。
* **時間保護：** 若進場後 2 日內未觸發停損/停利，於 第 3 日 (T+3) 強制平倉

**模型預測性能**：
* **準確率**： `59.91%` — 於測試集中展現穩定的趨勢方向預測力。
* **精準度**： `62.84%` — 有效過濾虛假訊號，確保進場品質。
* **召回率**： `83.51%` — 成功捕捉市場中絕大多數具備統計意義的獲利機會。

**回測結果**：
| 指標項目 | 統計結果 | 說明 |
| :--- | :--- | :--- |
| **總買進次數** | **41 次** | 展現良好的資金周轉率與付現能力 |
| **策略勝率 (Win Rate)** | **56.63%** | 優於隨機預測，具備穩定盈利能力 |

**未來買點優化與修正**：
* **執行價格調整**：目前回測以**當日收盤價**為進場基準，未來預計修正進場點為**翌日開盤價**，以更精準模擬真實交易中的情況與執行可行性。
* **投票平均機制**：計畫引入**投票機制**透過多組獨立訓練之模型進行多數決或機率平均，藉此沖銷類神經網路隨機初始化帶來的不確定性。
## 模組化設計:
專案採工業級模組化架構，展現良好的程式維護性：
* `crawler.py`: 數據獲取模組。
* `prepare_data.py`: 核心特徵工程與資料清理。
* `model.py`: 定義 LSTM 神經網路架構。
* `train_model.py`: 訓練邏輯與權重存檔。
* `evaluate_model.py`: 評估系統與訊號產出。
* `backtest.py`: 策略回測與模擬。
## 安裝與環境需求:
本專案開發環境基於 **Python 3.8+**，並使用以下核心套件：
* `FinMind`
* `PyTorch`
* `Pandas`
* `NumPy`
* `tqdm`
* `Matplotlib`
## 快速使用:
* 使用 `crawler.py` 抓取目標個股數據，預設支援 FinMind 資料源，可自由切換不同代號進行回測。
* 使用 `prepare_data.py` 進行特徵工程與資料分割，內建 **split** 函數可依據股票是否經拆分使用。
* 使用 `main.py` 進行訓練與預測，提供參數調校介面，使用者可針對不同市場環境調整訓練圈數、學習率、信心邊界值等。
